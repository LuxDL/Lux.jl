130 """
131 reverse(s::AbstractString) -> AbstractString
132
133 Reverses a string. Technically, this function reverses the codepoints in a string and its
134 main utility is for reversed-order string processing, especially for reversed
135 regular-expression searches. See also [`reverseind`](@ref) to convert indices in `s` to
136 indices in `reverse(s)` and vice-versa, and `graphemes` from module `Unicode` to
137 operate on user-visible "characters" (graphemes) rather than codepoints.
138 See also [`Iterators.reverse`](@ref) for
139 reverse-order iteration without making a copy. Custom string types must implement the
140 `reverse` function themselves and should typically return a string with the same type
141 and encoding. If they return a string with a different encoding, they must also override
142 `reverseind` for that string type to satisfy `s[reverseind(s,i)] == reverse(s)[i]`.
143
144 # Examples
145 ```jldoctest
146 julia> reverse("JuliaLang")
147 "gnaLailuJ"
148 ```
149
150 !!! note
151 The examples below may be rendered differently on different systems.
152 The comments indicate how they're supposed to be rendered
153
154 Combining characters can lead to surprising results:
155
156 ```jldoctest
157 julia> reverse("ax̂e") # hat is above x in the input, above e in the output
158 "êxa"
159
160 julia> using Unicode
161
162 julia> join(reverse(collect(graphemes("ax̂e")))) # reverses graphemes; hat is above x in both in- and output
163 "ex̂a"
164 ```
165 """
166 function reverse(s::Union{String,SubString{String}})::String
167 # Read characters forwards from `s` and write backwards to `out`
168 out = _string_n(sizeof(s))
169 offs = sizeof(s) + 1
170 for c in s
171 offs -= ncodeunits(c)
172 __unsafe_string!(out, c, offs)
173 end
174 return out
175 end
